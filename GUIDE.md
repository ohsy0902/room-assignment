# 🏢 기숙사 방 배치 시스템 상세 사용 설명서 (User Guide)

이 프로그램은 학생들의 성별, 학년, 선호하는 친구(동실 희망), 그리고 피해야 할 친구(기피 대상) 등의 복잡한 조건을 종합적으로 반영하여 **최적의 기숙사 방 배치를 자동으로 수행하고, 마우스 드래그를 통해 세밀하게 조정**할 수 있는 관리자용 도구입니다.

---

## 📂 1. 데이터 파일 준비 (CSV 양식)

자동 배치를 수행하기 위해 먼저 엑셀이나 스프레드시트 프로그램에서 아래 양식대로 데이터를 작성하고, **`CSV (쉼표로 분리) (*.csv)`** 형식으로 저장하여 준비해 주세요.

### ① 학생 명렬표 (필수)
배치 대상인 전체 학생 목록입니다.
- **파일명 권장**: `students.csv`
- **필수 열(헤더)**: `이름`, `학년`, `성별`
- **작성 규칙**: 성별은 `남` 또는 `여`로 정확히 기재해야 층/호실 성별 제한에 걸리지 않습니다.
- **예시**:
  ```csv
  이름,학년,성별
  홍길동,1,남
  성춘향,2,여
  이몽룡,3,남
  ```

### ② 선호학생 목록 (선택)
같은 방에 우선적으로 함께 배치하고 싶은 학생 목록입니다. 2명부터 최대 4명까지 한 행에 묶어 그룹을 구성할 수 있습니다.
- **파일명 권장**: `preferred.csv`
- **필수 열(헤더)**: `학생1`, `학생1학년`, `학생2`, `학생2학년`, `학생3`, `학생3학년`, `학생4`, `학생4학년`
- **작성 규칙**: 동명이인이 존재하는 경우를 대비해 **반드시 학년 열을 함께 기재**해 주세요. 학년이 기재되지 않은 경우 이름 매칭이 부정확해질 수 있습니다.
- **예시**:
  ```csv
  학생1,학생1학년,학생2,학생2학년,학생3,학생3학년
  홍길동,1,이몽룡,1,,
  성춘향,2,심청,2,황진이,2
  ```

### ③ 기피학생 목록 (선택)
같은 방에 배치되면 절대 안 되는 학생 쌍입니다.
- **파일명 권장**: `avoided.csv`
- **필수 열(헤더)**: `학생이름`, `학생학년`, `기피학생1`, `기피학생1학년`, `기피학생2`, `기피학생2학년`
- **작성 규칙**: 특정 학생이 기피하는 대상(기피학생 1, 2)을 한 줄에 나열합니다. 마찬가지로 학년 정보를 함께 적어야 동명이인 오류를 예방할 수 있습니다.
- **예시**:
  ```csv
  학생이름,학생학년,기피학생1,기피학생1학년
  홍길동,1,임꺽정,1
  ```

## ⚙️ 2. 화면 구성 요소 및 사전 설정

프로그램 상단 및 좌측의 제어판을 활용해 배치 전 기숙사 층별/호실별 규칙을 상세하게 정의할 수 있습니다.

### ① 기숙사별 탭 선택
* 화면 중앙 상단에 있는 **[알바트로스 / 베리타스 / 명덕재]** 탭을 클릭하여 각 건물별 배치 현황을 전환할 수 있습니다.

### ② 층 성별 설정 (Floor Gender Restrictions)
* 각 층의 전용 성별을 한 번에 설정합니다.
* 층 선택 옆의 **[제한 없음 / 남학생 / 여학생]** 버튼을 통해 설정합니다.
* 특정 성별 전용으로 설정하면, 성별이 맞지 않는 학생은 드래그 앤 드롭 수동 배치나 자동 배치 시 해당 층에 절대 배정되지 않습니다.

### ③ 층 레이아웃 일괄 설정
* 층 전체 호실에 동일한 학년 구성을 한 번에 적용합니다.
* **[👥 층 레이아웃 일괄 설정]** 버튼을 눌러 나타나는 모달 창에서 프리셋(예: 1학년 2명 + 2학년 2명)을 고르거나 직접 학년 블록을 배치한 뒤 저장하면, 해당 층의 모든 호실에 동일한 학년별 정원 제한 규칙이 복사됩니다.

### ④ 호실별 개별 설정 (방 카드 클릭)
* **호실 학년 레이아웃 설정**: 특정 기숙사 방 카드의 빈 영역을 클릭하면 해당 방만 개별적으로 학년 레이아웃을 바꿀 수 있는 설정 창이 열립니다. (예: 101호는 3학년 전용방, 102호는 1학년 3명 + 2학년 1명 등)
* **호실 미사용 지정 (Unused)**: 방 카드 우측 상단의 **[사용 중 / 미사용]** 토글 스위치를 클릭해 기숙사 수리나 청소 등으로 비워둬야 할 호실을 설정할 수 있습니다. 미사용 호실은 어두운 회색으로 변하며 배치 대상에서 제외됩니다.

## 🎲 3. 자동 배치 알고리즘 작동 원리

이 시스템의 자동 배치는 다음과 같은 정교한 알고리즘 단계를 거쳐 균등하고 효율적으로 이루어집니다.

### 1단계: 선호학생 그룹 배정 (무작위 분산)
* 선호학생 파일에 지정된 그룹(2~4인)을 먼저 배정합니다.
* **최적화된 분산 배정**: 특정 방에 여러 선호 그룹이 뭉쳐서 2쌍씩 기계적으로 묶이지 않도록, 선호학생 배정 시에는 **사용 가능한 호실 목록을 무작위(Shuffle)로 섞어서 첫 방을 할당**합니다. 이로 인해 기숙사 전반에 선호 그룹이 자연스럽게 골고루 흩어지며, 확률에 따른 자연스러운 랜덤 배치가 보장됩니다.

### 2단계: 나머지 학생 배정 (미사용 호실 최소화)
* 선호그룹 배정이 끝난 뒤, 남은 모든 학생들을 셔플하여 공평하게 순차 배정합니다.
* **압축 밀집 정렬 알고리즘**: 이미 학생이 들어가 있는 방을 우선적으로 꽉 채워 배정하도록 방 정렬 기준이 작동합니다. 이 방식을 통해 빈자리가 분산되는 것을 막고 **완전히 비어 있는 빈 방(공실)의 개수를 최대로 확보**하여 기숙사 운영 및 에너지 관리 효율을 극대화합니다.

## ⚡ 4. 수동 조정 및 예외 처리

### ① 드래그 앤 드롭 (Drag & Drop)
* **방에 배정**: 우측 사이드바의 `미배치 학생` 리스트에서 학생 카드를 집어 원하는 호실 카드로 드래그합니다.
* **방에서 퇴실**: 방 카드 안에 있는 학생 카드를 집어 다시 우측의 `미배치 학생` 영역으로 놓으면 배정이 취소됩니다.
* **방 간 이동**: 방 내부의 학생 카드를 다른 방 카드로 직접 옮겨 배치할 수 있습니다.

### ② 강제 배치 모달 및 예외 알림
* 자동 배치 도중 기피 관계(avoided.csv) 충돌이나 학년 레이아웃 정원 초과 등으로 인해 배치가 불가능한 학생이 발생하면 **[강제 배치 예외창]**이 나타납니다.
  * **건너뛰기**: 배치 조건을 유지한 채 해당 학생을 배정하지 않고 미배치 목록으로 뺍니다.
  * **강제 배치**: 기피 관계를 무시하고 남은 빈자리에 억지로 밀어 넣습니다. 강제 배치된 학생들의 카드에는 **빨간색 느낌표(!) 경고 마크**가 붙어 관리자가 한눈에 파악할 수 있습니다.

## 💾 5. 데이터 보존 및 엑셀 다운로드

### ① 실시간 브라우저 세션 보존
* 프로그램 동작 중 새로고침을 하거나, 웹 브라우저 창을 완전히 닫고 컴퓨터를 껐다 켜더라도 **이전에 설정한 기숙사 규칙, 업로드한 파일명 정보, 학생들의 방 배치 완료 상태가 그대로 복구**됩니다.
* 이는 로컬 웹 저장소(`localStorage`)에 실시간으로 작업 내역이 안전하게 세이브되기 때문입니다.

### ② 엑셀로 내보내기 (Export)
* 배치가 끝나면 우측 상단의 **[📊 Excel로 내보내기]** 버튼을 클릭합니다.
* **학년순 + 이름순 정렬 탑재**: 엑셀 표로 다운로드할 때, 기숙사 호실 카드 화면과 동일하게 호실 내 학생들이 **학년순(오름차순)으로 먼저 정렬되고, 같은 학년 내에서는 이름 가나다순(오름차순)으로 정렬**되어 저장됩니다. 기숙사 배정 장부 및 호실 대조 대장으로 활용 시 행정 가독성이 극대화됩니다.

### ③ 배치 초기화 (Reset)
* 처음부터 기숙사 설정을 다시 조정하여 배치를 돌리고 싶다면 **[🔄 초기화]** 버튼을 누릅니다. 업로드했던 파일 정보(학생 명렬 등)는 그대로 유지된 채, 기숙사 방에 들어가 있던 모든 학생만 미배치 상태로 원복됩니다.
