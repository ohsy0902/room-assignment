# 🏢 기숙사 방 배치 시스템 사용 설명서 (User Guide)

이 프로그램은 학생들의 성별, 학년, 선호하는 친구, 그리고 피해야 할 친구(기피 대상) 등의 복잡한 조건을 반영하여 **최적의 기숙사 방 배치를 자동으로 수행하고 수동으로 세밀하게 조정**할 수 있는 웹 기반 도구입니다.

---

## 📂 1. 사전 준비 (CSV 파일 양식)

배치를 시작하기 전, 아래의 세 가지 CSV 파일을 준비해야 합니다. (엑셀 등에서 작성 후 `CSV (쉼표로 분리) (*.csv)` 형식으로 저장하시면 됩니다.)

### ① 학생 명렬표 (필수)
기숙사에 배치할 전체 학생 목록입니다.
- **파일명 예시**: `students.csv`
- **필수 컬럼(헤더)**: `이름`, `학년`, `성별`
- **예시**:
  ```csv
  이름,학년,성별
  홍길동,1,남
  성춘향,2,여
  이몽룡,3,남
  ```

### ② 선호학생 목록 (선택)
같은 방에 우선적으로 함께 배치하고 싶은 학생 그룹입니다. (2인~최대 4인까지 그룹 구성 가능)
- **파일명 예시**: `preferred.csv`
- **컬럼(헤더)**: `학생1`, `학생1학년`, `학생2`, `학생2학년`, `학생3`, `학생3학년`, `학생4`, `학생4학년`
  *(동명이인이 있을 수 있으므로 학년 컬럼을 기재하는 것을 권장합니다.)*
- **예시**:
  ```csv
  학생1,학생1학년,학생2,학생2학년,학생3,학생3학년
  홍길동,1,이몽룡,1,,
  성춘향,2,심청,2,황진이,2
  ```

### ③ 기피학생 목록 (선택)
서로 같은 방에 배치되면 안 되는 기피 대상 지정 파일입니다.
- **파일명 예시**: `avoided.csv`
- **컬럼(헤더)**: `학생이름`, `학생학년`, `기피학생1`, `기피학생1학년`, `기피학생2`, `기피학생2학년`
- **예시**:
  ```csv
  학생이름,학생학년,기피학생1,기피학생1학년
  홍길동,1,임꺽정,1
  ```

## ⚙️ 2. 화면 구성 및 설정

프로그램 상단과 좌측 사이드바를 활용해 다양한 사전 규칙을 정의할 수 있습니다.

### ① 층 성별 설정 (Dormitory Gender)
각 층 전체의 성별 전용 여부를 한 번에 설정합니다.
- 상단 탭에서 기숙사(알바트로스 / 베리타스 / 명덕재)를 선택하고 층을 고른 뒤, **남학생 전용 / 여학생 전용 / 제한 없음** 버튼을 클릭하여 적용할 수 있습니다.
- 특정 성별 전용으로 설정하면, 성별이 맞지 않는 학생은 드래그 앤 드롭이나 자동 배치로도 들어갈 수 없습니다.

### ② 호실 미사용 지정 (Unused Room)
기숙사 방 카드 우측 상단의 **사용 중 / 미사용** 토글 버튼을 클릭하여 수리 중이거나 비워두어야 하는 방을 `미사용`으로 지정할 수 있습니다. 
- 미사용 호실에는 자동 배치 및 수동 배치가 모두 금지됩니다.

### ③ 호실 학년 레이아웃 구성 (Room Layout)
특정 학년들을 섞어서 배정하거나 고정된 구성(예: 1학년 2명 + 2학년 2명)으로 배정하고 싶을 때 사용합니다.
- 기숙사 방 카드의 빈 공간을 클릭하면 **레이아웃 설정 창**이 나타납니다.
- 드래그하거나 학년 블록을 클릭하여 원하는 학년 구성을 채워 넣은 뒤 저장하면, 해당 방에는 지정된 학년의 슬롯에만 학생이 들어올 수 있습니다. (예: 1학년 전용 방, 1학년-2학년 혼합 방 등)

## ⚡ 3. 배치 실행 및 수동 조정

### ① 자동 배치 실행
1. 준비된 CSV 파일 3개를 화면 좌측의 파일 업로드 칸에 각각 업로드합니다. (업로드 성공 시 초록색으로 파일명이 나타납니다.)
2. 배치를 적용할 범위(**전체 기숙사 / 특정 기숙사 / 특정 층**)를 설정합니다.
3. **[자동 배치 시작]** 버튼을 클릭합니다.

> 💡 **미사용 방 최소화 알고리즘 탑재**
> 본 시스템은 빈 방이 산발적으로 생기지 않도록, **이미 학생이 배치된 방을 우선적으로 꽉 채워 배정**하는 정렬 알고리즘을 사용합니다. 이를 통해 완전히 사용되지 않는 '미사용 빈 방'의 개수가 최대화되며, 남는 방을 효율적으로 관리할 수 있습니다.

### ② 강제 배치 모달
기피 대상 조건 등으로 인해 자동 배치가 불가능한 예외 학생이 발생하는 경우, 경고 모달이 뜹니다.
- **건너뛰기**: 배치되지 못한 학생은 좌측 `미배치 학생 현황` 사이드바 목록으로 빠지며, 사용자가 수동으로 처리합니다.
- **강제 배치**: 기피 조건을 무시하고 남은 자리에 강제로 배치합니다. 이 경우 배치된 학생 카드에 **느낌표(!) 경고 배지**가 표시됩니다.

### ③ 수동 드래그 앤 드롭 조정
- **방에 배정**: 좌측 사이드바의 미배치 학생을 드래그하여 원하는 호실 카드로 놓습니다.
- **방에서 퇴실**: 호실 안에 있는 학생 카드를 드래그하여 다시 좌측 `미배치 학생 현황` 사이드바 영역으로 떨어뜨리면 배정이 해제됩니다.
- **방 간 이동**: 방에 이미 들어가 있는 학생 카드를 다른 방으로 직접 드래그하여 이동할 수 있습니다.

## 💾 4. 데이터 보존 및 결과 내보내기

### ① 브라우저 자동 저장 (새로고침 보존)
- 웹 브라우저를 **새로고침하거나 창을 완전히 종료했다가 다시 켜더라도** 이전에 업로드한 파일명, 기숙사 설정, 학생들의 방 배치 현황이 고스란히 복구됩니다.
- 이는 브라우저의 `localStorage` 안전 영역을 활용하므로 별도의 저장 버튼을 매번 누르지 않아도 모든 작업이 실시간으로 저장됩니다.

### ② 엑셀로 내보내기 (Export)
- 배치가 완료되면 우측 상단의 **[Excel 내보내기]** 버튼을 누릅니다.
- 현재의 배치 완료 상태(기숙사, 층, 호실, 배치된 학생 정보)가 정리된 깔끔한 엑셀(.xlsx) 문서로 다운로드됩니다.

### ③ 초기화
- 처음부터 다시 배치하고 싶다면 **[배치 초기화]** 버튼을 클릭합니다. 업로드된 파일 정보는 유지한 채 학생들의 호실 배정 상태만 모두 해제됩니다.
